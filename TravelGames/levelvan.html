HOW TO USE

Getting set up

1
SHARE THE APP LINK
Before you arrive at the campsite, send the LevelVan link to everyone who needs it — via AirDrop, WhatsApp or iMessage. Both the caravan phone and the driver's phone need the page open in their browser. No app download needed.
2
PLACE THE SENSOR PHONE
Put one phone flat inside the caravan — on a table, bench or floor. Point the top of the phone toward the hitch end (the front of the caravan, where it connects to the car). This is important so pitch and roll read correctly.
3
TAP SENSOR
On the caravan phone, tap SENSOR. On iOS, tap Allow Motion when prompted — this lets the phone read its tilt. A 4-digit PIN will appear at the top of the screen.
4
CONNECT THE DISPLAY
On the driver's phone, tap DISPLAY and type in the 4-digit PIN from the sensor phone. The connection happens automatically on the 4th digit. You can also tap 📤 SHARE on the sensor to send a link that connects instantly.
5
LEVEL THE CARAVAN
The display shows a top-down caravan that tilts in real time. The coloured dot shows where the high side is — green means level, orange means close, red means adjust.

Left / Right (Roll): Drive the car forward or back onto leveling blocks under the low wheels until the dot centres left-right.

Front / Back (Pitch): Once roll is sorted, use the jockey wheel to raise or lower the hitch until the dot centres front-to-back.

Good to know

📺 MULTIPLE DISPLAYS
Any number of phones can connect to the same sensor at once. Just share the PIN or the link. Useful if the driver and a helper both want to watch the level.
🔄 RECONNECTING
If a display disconnects, just tap DISPLAY again and re-enter the PIN. The sensor keeps running and the PIN stays the same until you go back to the home screen.
⚠️ PHONE PLACEMENT
Make sure the sensor phone is lying completely flat and won't slide around while you're moving the caravan. Even a small tilt in the phone's resting position will offset the readings.

Pairing

🔗 PERMANENT PAIRING
Once paired, both phones remember each other. Next time you open the app, they'll automatically reconnect — no PIN needed. Screens stay on automatically while leveling.

How it connects

LevelVan uses WebRTC — a direct peer-to-peer data channel between the two phones. Once connected, tilt data flows directly between devices over your local network at ~15 times per second. No server is involved in the data stream.
📶 Same WiFi network
Both phones on the same home or campsite WiFi. This is the easiest setup — everything works automatically. The initial handshake uses the PeerJS relay server briefly, then data flows directly over your local network.
🌐 Internet available (mobile data)
If both phones have mobile data (even on different networks), the WebRTC handshake completes via the internet and then the connection is established. Works well in most campgrounds with any signal.
🏔️ No signal — remote location
No WiFi, no mobile data? Use a phone hotspot:
  1. On the Display phone (driver's phone), turn on a Personal Hotspot.
  2. On the Sensor phone (caravan phone), connect to that hotspot via WiFi settings.
  3. Now both phones are on the same local network — open LevelVan on both and connect as normal.
  4. The display phone creates the hotspot and runs the display — this works fine.

Note: The very first connection still needs a brief internet handshake. If there is truly zero signal, try the hotspot approach — the hotspot phone itself often still has enough signal to complete the handshake even when the other phone doesn't.

Technical details

🔢 PIN SYSTEM
The sensor generates a random 4-digit PIN which maps to a unique PeerJS peer ID (levelvan-XXXX-sensor). The display connects directly to that peer ID. The PIN changes every session.
📺 MULTIPLE DISPLAYS
Each display gets its own unique peer ID (levelvan-XXXX-display-[timestamp]). The sensor accepts unlimited simultaneous connections. All displays receive the same live data stream.
📐 SENSOR DATA
Uses the DeviceOrientation API — gamma for roll (left/right) and beta for pitch (front/back). iOS requires explicit permission. Data is sent at ~15Hz (every 66ms) to balance responsiveness and battery.
⚡ RELAY SERVER
The initial WebRTC handshake uses 0.peerjs.com (free public PeerJS server). This is only needed for the first few seconds to exchange connection details. After that, all data is peer-to-peer. The relay server never sees your tilt data.
